# 常见问题

## 录音相关

### 录音没有声音 / 转写结果为空

1. 检查麦克风是否被系统识别
2. 确认录音工具已安装（Linux: `parecord`, Windows: `ffmpeg`）
3. 查看日志输出：`RUST_LOG=altgo=debug altgo`

### ffmpeg 录音启动很慢

ffmpeg 的 `dshow` 设备初始化需要时间。首次启动时会自动预热设备缓存，后续录音会更快。

## 按键相关

### 按右 Alt 没有反应

import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';

<Tabs groupId="os">
<TabItem value="linux" label="Linux">

1. **用户是否在 `input` 组并已重新登录？** 无法读取 `/dev/input/event*` 时，Linux 按键监听会失败：`sudo usermod -aG input "$USER"` 后注销/重启再试。
2. 确认 `xinput` 可用：`which xinput`（X11 路径）。
3. 检查 Alt 绑定：`xmodmap -pke | grep Alt`；配置里默认 `key_name = "Alt_R"`，与设置页「按下以设置」一致即可。
4. **Wayland**：仍依赖能访问 evdev；若仅用 XWayland 且无权限，现象与未加 `input` 组类似。剪贴板在 Wayland 上建议安装 **`wl-copy`**（或 `xclip`/`xsel` 经 XWayland）。

</TabItem>
<TabItem value="windows" label="Windows">

1. 使用打包版时依赖已随包提供；从源码跑时请确认 `make deps-windows` 等与 README 一致。
2. 若使用 IME，可适当增大 `min_press_duration_ms` 减轻误触。

</TabItem>
</Tabs>

### 误触发 / IME 冲突

增加 `min_press_duration_ms` 的值（默认 100ms）来过滤短按抖动：

```toml
[key_listener]
min_press_duration_ms = 200
```

## 转写相关

### 中文识别效果不好

1. 使用更大的模型（推荐 `medium` 或 `large`）
2. 确认 `language = "zh"` 设置正确
3. 如果使用 API 模式，可以设置 `prompt` 提供领域词汇

```toml
[transcriber]
language = "zh"
prompt = "技术会议，产品讨论"
```

### 本地模型在哪里下载？

从 Hugging Face 下载 GGML 格式模型：

| 模型 | 大小 | 中文效果 |
|------|------|---------|
| tiny | ~75 MB | 一般 |
| base | ~148 MB | 日常够用 |
| small | ~488 MB | 较好 |
| medium | ~1.5 GB | 推荐 |
| large | ~3 GB | 最佳 |

下载地址：`https://huggingface.co/ggerganov/whisper.cpp/resolve/main/ggml-{size}.bin`

## 润色相关

### 润色太慢

1. 降低润色级别（`heavy` → `medium` → `light`）
2. 使用更快的模型或本地 Ollama
3. 设置 `level = "none"` 关闭润色

### 润色改变了原意

使用 `light` 级别，只修正标点和错别字，不改变内容。或者使用自定义 system prompt：

```toml
[polisher]
level = "light"
system_prompt = "只修正标点和错别字，不要改变原文的意思和语气"
```

## 剪贴板与悬浮窗

### 提示成功但粘贴出来是旧内容 / 剪贴板失败

1. **Linux**：安装 `xclip`、`xsel` 或 `wl-copy` 之一，并确保在图形会话内运行 altgo。
2. 在**悬浮窗**或**历史**页使用复制按钮（WebView 手势复制有时比后端剪贴板更稳）。
3. 调试：`RUST_LOG=altgo=debug altgo` 查看是否有 clipboard 相关错误。

## 通用

### 如何后台运行？

<Tabs groupId="os">
<TabItem value="linux" label="Linux">

```bash
nohup altgo &
# 或使用 systemd user service
```

</TabItem>
<TabItem value="windows" label="Windows">

```powershell
Start-Process -WindowStyle Hidden .\altgo.exe
```

</TabItem>
</Tabs>

### 如何查看日志？

```bash
RUST_LOG=altgo=debug altgo
```

可用的日志级别：`debug`、`info`、`warn`、`error`
