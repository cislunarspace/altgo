# 快速开始

文档以 **Linux** 为主：本软件目前仅在 **Ubuntu 20.04** 上做过测试与部署验证；其他系统未保证。Windows 用户请优先使用 [Releases](https://github.com/cislunarspace/altgo/releases) 中的 **MSI** 或 zip。

**官方预编译包**（deb / AppImage / MSI 等）会尽量把 **ffmpeg**、**whisper-cli** 等与程序一并打包，安装或解压后**开箱即用**，无需再为录音、转写单独安装这些二进制。

## Linux：安装前必做（input 组）

**必须**将当前用户加入 `input` 组，否则无法读取 `/dev/input/event*`，按键监听会失败。执行后**须重新登录**：

```bash
sudo usermod -aG input "$USER"
```

其余与桌面、音频、通知相关的系统库，**`.deb` 会通过依赖声明自动拉取**；若安装器提示缺包，按提示补装即可，不必自行对照长清单。

## 最终用户：装好后在「设置」里完成一切

1. 从 [Releases](https://github.com/cislunarspace/altgo/releases) 安装 **`.deb`** 或使用 **AppImage**（`chmod +x` 后运行）。
2. 完成上文 **input 组**。
3. 启动应用，打开 **设置**：
   - 在 **转写** 中选用 **本地** 或 **API**（若需要），下载并启用模型，或填写高级路径；
   - 按需配置 **润色**、**外观**、**触发键**（可用「按下以设置」）；
   - 点击 **保存**。
4. 识别结果在**悬浮窗**中展示，由你自行选择复制。

**不要**把「先复制 `configs/altgo.toml` 再手改」当作默认路径——那是高级用法。日常用户只使用界面即可。

## 开发者：从源码构建

与主仓库一致，使用 **`make deps-linux`** / **`make deps-windows`** 拉取依赖，再用 **`make build`** 打包。**日常验证也建议以 `make build` 为主**。

```bash
git clone https://github.com/cislunarspace/altgo.git
cd altgo
cd frontend && npm install && cd ..
make deps-linux
make build
```

若需快速改前端 UI，可临时使用 `cargo tauri dev`；完整链路仍以 `make build` 为准。

## 高级：仅当你要手改配置文件时

配置文件路径：**Linux** `~/.config/altgo/altgo.toml`，**Windows** `%APPDATA%\altgo\altgo.toml`。字段说明见主仓库 [`configs/altgo.toml`](https://github.com/cislunarspace/altgo/blob/master/configs/altgo.toml)。

## 启动

运行 `altgo` 或安装目录下的可执行文件。按住 **右 Alt** 说话，松开后在**悬浮窗**查看结果；触发键在 **设置** 中修改。
